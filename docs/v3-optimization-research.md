## 🔍 各优化点详细设计

### 1. 向量预过滤 + LLM 二级门控

**流程**:
```
子任务结果 → 嵌入 → 计算与查询的余弦相似度 →
  相似度 < 阈值 → 直接过滤，不需要 LLM 门控 →
  相似度 > 阈值 → 送 LLM 计算门控分数
```

**配置默认值**:
```python
enable_vector_prefilter = True
vector_similarity_threshold = 0.3
```

### 2. JSON 压缩门控输出

**当前**: LLM 输出自然语言解释 + JSON，浪费很多 Token

**改进**: 只输出 JSON，不需要解释:

```
### 请你给每个子任务结果打分，输出JSON格式：
{
  "task_scores": [
    {
      "task_id": "task_1",
      "score": 0.85
    },
    {
      "task_id": "task_2", 
      "score": 0.45
    }
  ]
}
```

预期节省 **30-50%** 门控 Token。

### 3. 累积增益早停

**新增逻辑**:
```python
cumulative_gain = sum(level.gate_score for level in processed_levels)
if cumulative_gain >= cumulative_gain_threshold:
    # 已经获得足够信息，提前停止
    break
```

**默认阈值**: `cumulative_gain_threshold = 2.5`  
（3 层最大是 3.0，所以一般 2 层就够了）

### 4. 合并小 Block

**改进分组逻辑**:
```
如果剩余子任务数 < block_size / 2:
    合并到上一个 Block
否则:
    新开 Block
```

避免出现很多只有 1-3 个子任务的小 Block，减少 Block 总数，从而减少聚合调用。

### 5. 赢者通吃 (Winner-Take-All) 后处理

**流程**:
```
侧抑制后 → 按语义聚类 → 每个簇只保留 Top-K 得分最高的结果 →
  其他结果抑制掉
```

**默认配置**: `wta_top_k = 3` 每个簇保留 3 个

进一步增强稀疏性，去除高度相似的冗余结果。

### 6. 激活区容量限制（7±2 法则）

**认知科学依据**: 人脑工作记忆容量约 7±2 个组块

**实现**:
```python
max_active_blocks = 7  # 最多 7 个 Block 同时激活
if len(active_blocks) > max_active_blocks:
    # 按门控分数排序，低分数挂起到结果区
    sort_by_gate_score → 挂起低分数的
```

避免激活区信息过载，保持处理焦点。
